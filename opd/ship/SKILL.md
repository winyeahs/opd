# 阶段五：部署与交付

**执行时间占比**：5-10%

## 核心任务

部署配置、发布说明、用户手册编写

## 输入文档

读取阶段四的交付物：
- 测试报告.md
- 质量评估报告.md
- 测试用例.md

以及阶段一的验收标准：
- 验收标准.md

## 交付物

**文档输出目录**：项目根目录的 `docs/阶段五-部署与交付/`

**参考模板**：技能包内 `templates/阶段五部署与交付/` 目录

生成以下内容：

1. **部署包** - 必需
   - 目录：`deploy/`
   - 部署脚本
   - 配置文件
   - 环境变量配置

2. **部署文档.md** - 必需
   - 目录：`docs/阶段五-部署与交付/`
   - 部署环境说明
   - 部署步骤
   - 配置说明
   - 常见问题处理

3. **发布说明.md** - 必需
   - 目录：`docs/阶段五-部署与交付/`
   - 版本号
   - 新功能列表
   - 已修复问题
   - 已知问题
   - 升级注意事项

4. **用户手册.md** - 推荐
   - 目录：`docs/阶段五-部署与交付/`
   - 功能说明
   - 使用指南
   - 常见问题

5. **运维手册.md** - 推荐
   - 目录：`docs/阶段五-部署与交付/`
   - 系统监控
   - 日志管理
   - 备份恢复
   - 故障处理

## 执行流程

1. **读取上游文档**：读取阶段四的测试报告和质量评估报告
2. **发布决策**：根据质量评估报告决定是否发布
3. **准备部署包**：打包应用、配置文件、部署脚本
4. **编写部署文档**：详细说明部署步骤和配置
5. **编写发布说明**：记录版本变更内容
6. **编写用户手册**：编写用户使用指南
7. **编写运维手册**：编写运维操作指南
8. **质量门禁检查**：验证发布就绪状态

## 质量门禁

**阶段五门禁**：发布就绪
- [ ] 部署文档完整
- [ ] 发布说明清晰
- [ ] 用户手册完备
- [ ] 回滚方案明确
- [ ] 监控配置完成

## 发布检查清单

**发布前检查**：
- [ ] 所有严重缺陷已修复
- [ ] 测试报告显示通过
- [ ] 质量评估报告同意发布
- [ ] 部署包已准备完毕
- [ ] 回滚方案已制定

**部署后验证**：
- [ ] 系统功能正常
- [ ] 性能指标达标
- [ ] 监控告警正常
- [ ] 日志记录正常

## 与上下游关系

- **上游**：读取 `docs/阶段四-测试与验证/` 目录的测试报告和质量评估报告
- **下游**：阶段六（retro）读取 `docs/阶段五-部署与交付/` 目录的发布说明和运维手册
- **回溯验证**：确保交付内容符合验收标准

## 使用示例

```
/opd:ship 发布v1.0版本
/opd:ship 部署到生产环境
/opd:ship 准备发布包和文档
```

## 回滚策略

发布前必须准备回滚方案：

1. **版本回滚**：
   - 保留上一个版本的部署包
   - 准备快速回滚脚本
   - 记录回滚步骤

2. **数据回滚**：
   - 数据库变更前备份
   - 准备数据回滚脚本
   - 测试回滚脚本有效性

3. **配置回滚**：
   - 记录配置变更内容
   - 保存原配置文件
   - 准备配置恢复方案

## 注意事项

- 发布前确保所有测试通过
- 部署文档要详细清晰，便于操作
- 发布说明要准确记录变更内容
- 用户手册要通俗易懂
- 运维手册要包含故障处理流程
- 回滚方案必须经过测试验证
