# 阶段三：开发与实现

**执行时间占比**：40-50%

> ⚠️ **重要提醒：长时间执行不可中断**
> 
> 本阶段是整个开发流程中耗时最长的阶段（40-50%），涉及大量代码编写、调试和测试工作。
> 
> **执行要求**：
> - 执行过程中**严禁中断**，必须保持连续性
> - 确保有充足的时间窗口（建议预留完整的工作时段）
> - 中断可能导致：代码状态不一致、上下文丢失、重复工作
> - 如需暂停，必须记录当前进度和待完成事项，以便恢复后继续
> 
> **最佳实践**：一次性完成一个功能模块的开发、测试和审查，避免碎片化执行。

## 核心任务

代码开发、单元测试、代码审查

## 核心方法论：OWASP Top 10安全检查

在代码完成后必须执行OWASP Top 10安全扫描：

| 编号 | 安全风险 | 检查要点 |
|------|----------|----------|
| A01 | 访问控制失效 | 身份认证、权限校验、会话管理 |
| A02 | 加密失败 | 敏感数据加密、传输安全、密钥管理 |
| A03 | 注入攻击 | SQL注入、XSS、命令注入防护 |
| A04 | 不安全设计 | 业务逻辑漏洞、权限边界设计 |
| A05 | 安全配置错误 | 默认配置、错误信息泄露、未授权功能 |
| A06 | 脆弱组件 | 第三方依赖版本、已知漏洞检查 |
| A07 | 身份认证失败 | 密码策略、多因素认证、会话失效 |
| A08 | 数据完整性失败 | 输入验证、签名校验、防篡改 |
| A09 | 日志监控失败 | 安全日志、异常告警、审计追踪 |
| A10 | SSRF | 外部资源访问控制、URL验证 |

**执行要点**：
- 代码完成后立即执行安全扫描
- 发现安全漏洞必须修复后再继续
- 记录安全检查结果到代码审查报告

   - 目录：`tests/`

   - 目录：`docs/阶段三-开发与实现/`

   - 改进建议

## 执行流程

1. **读取上游文档**：读取阶段二的架构设计和技术设计文档
2. **环境搭建**：配置开发环境、依赖项
3. **编码实现**：按照架构设计实现功能代码
4. **单元测试**：编写单元测试，覆盖率≥80%
5. **安全扫描**：执行OWASP Top 10安全检查
6. **代码审查**：生成代码审查报告
7. **质量门禁检查**：验证代码质量和测试覆盖率
