# plan-review

**阶段二：设计与架构 - 设计审核**

---

## 技能概述

本技能用于设计与架构阶段，对架构设计、技术选型、接口设计进行系统性审核，并执行STRIDE威胁模型安全分析。

---

## 上游输入

读取阶段一产出的文档：

- 需求规格说明书.md
- 前端UI设计说明书.md
- 验收标准.md

---

## 执行流程

### 第一步：读取需求文档

读取并理解需求规格说明书，提取：

- 核心功能点
- 技术约束
- 性能要求
- 安全要求

### 第二步：架构设计审核

检查以下维度：

| 审核维度 | 检查项 |
|----------|--------|
| 架构合理性 | 分层清晰、职责单一、扩展性好 |
| 技术选型 | 成熟稳定、社区活跃、团队熟悉 |
| 性能设计 | 缓存策略、数据库优化、并发处理 |
| 可维护性 | 模块化程度、代码规范、文档完整 |

### 第三步：STRIDE威胁模型分析

对系统进行六种威胁类型分析：

| 威胁类型 | 英文 | 检查要点 |
|----------|------|----------|
| 仿冒 | Spoofing | 身份伪造、证书伪造、钓鱼防护 |
| 篡改 | Tampering | 数据完整性、防篡改机制 |
| 抵赖 | Repudiation | 操作日志、不可否认性证明 |
| 信息泄露 | Information Disclosure | 数据泄露、权限泄露、敏感信息暴露 |
| 拒绝服务 | Denial of Service | 资源耗尽防护、限流机制 |
| 权限提升 | Elevation of Privilege | 越权访问、权限边界 |

### 第四步：输出设计文档

生成以下交付物：

- 架构设计文档.md
- 前端技术设计说明书.md
- 后端技术设计说明书.md
- 接口设计说明书.md
- 数据库设计.sql
- 安全方案文档.md
- 设计审核报告.md

## 质量门禁

进入下一阶段前必须满足：

- 架构设计合理，分层清晰
- 技术选型有明确理由
- STRIDE威胁分析完成
- 接口设计文档完整
- 设计审核报告通过
