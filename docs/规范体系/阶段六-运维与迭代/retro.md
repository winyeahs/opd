## 执行流程

### 第一步：系统监控

持续监控系统运行状态：

| 监控维度 | 监控指标 |
|----------|----------|
| 性能监控 | 响应时间、吞吐量、资源占用 |
| 错误监控 | 错误率、异常日志、告警统计 |
| 用户监控 | 用户活跃度、功能使用率、转化率 |
| 安全监控 | 安全事件、异常访问、漏洞报告 |

### 第二步：问题分析

发现问题后的分析流程：

```
1. 记录问题到问题分析报告
2. 分析根因（5 Whys 方法）
3. 评估影响范围
4. 制定修复方案
5. 记录到迭代计划
```

### 第三步：验收标准回溯

回溯阶段一的验收标准：

- 已交付功能是否满足验收条件
- 是否偏离原始需求
- 是否有遗漏功能

**闭环验证**：
- 满足验收 → 项目完成
- 偏离需求 → 记录到迭代计划
- 遗漏功能 → 记录到迭代计划

### 第四步：迭代规划

制定下一迭代计划：

| 规划维度 | 内容 |
|----------|------|
| 新功能 | 下一迭代要开发的功能 |
| Bug修复 | 待修复的问题清单 |
| 优化项 | 性能优化、体验优化 |
| 资源规划 | 时间、优先级、工作量 |

### 第五步：输出交付物

生成以下交付物：

- 运维报告.md
- 问题分析报告.md
- 迭代计划.md
- 优化建议.md
