# qa

**阶段四：测试与验证 - 真实浏览器QA**

---

## 技能概述

本技能用于测试与验证阶段，使用 Playwright 打开真实浏览器进行端到端测试，发现并修复问题。

---

## 上游输入

读取阶段三产出的文档：

- 测试用例.md
- 代码审查报告.md
- 验收标准.md

---

## 执行流程

### 第一步：读取测试文档

读取并理解测试计划和测试用例，提取：

- 测试场景
- 用户流程
- 验收条件
- 边界情况

### 第二步：启动真实浏览器

使用 Playwright 工具：

```
1. 打开 Chrome/Firefox/Safari 等真实浏览器
2. 访问 staging 或本地测试服务器
3. 设置浏览器窗口大小和视口
```

### 第三步：执行用户流程测试

按照测试用例执行：

| 测试类型 | 操作步骤 |
|----------|----------|
| 登录测试 | 打开登录页 → 输入账号密码 → 点击登录 → 验证跳转 |
| 功能测试 | 导航到功能页 → 执行操作 → 验证结果 |
| 边界测试 | 输入极端值 → 验证错误处理 |
| 流程测试 | 完整用户流程 → 验证端到端结果 |

### 第四步：截图记录

关键步骤截图留存：

- 每个页面加载后截图
- 每个操作执行后截图
- 发现问题时截图
- 修复后验证截图

### 第五步：自动修复

发现问题后的处理流程：

```
1. 记录问题到缺陷报告
2. 分析问题根因
3. 自动修复代码
4. 回归验证
```

### 第六步：输出测试报告

生成以下交付物：

- 测试计划.md
- 测试用例.md
- 缺陷报告.md
- 测试报告.md
- 质量评估报告.md

## 质量门禁

进入下一阶段前必须满足：

- 功能测试通过率100%
- 性能测试达标
- 安全测试通过
- 测试报告完整
- 无高严重程度缺陷遗留
