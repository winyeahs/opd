# develop

**阶段三：开发与实现 - 代码开发**

---

## 技能概述

本技能用于开发与实现阶段，按照架构设计文档进行代码开发，执行OWASP安全检查，并完成代码审查。

---

## 上游输入

读取阶段二产出的文档：

- 架构设计文档.md
- 前端技术设计说明书.md
- 后端技术设计说明书.md
- 接口设计说明书.md
- 数据库设计.sql

---

## 执行流程

### 第一步：读取设计文档

读取并理解架构设计，提取：

- 模块划分
- 接口定义
- 数据结构
- 技术规范

### 第二步：代码实现

按照设计文档编写代码：

1. **创建项目结构**：按照架构设计创建目录结构
2. **实现核心模块**：优先实现核心业务逻辑
3. **编写单元测试**：同步编写单元测试，覆盖率≥80%
4. **代码注释**：关键逻辑添加清晰注释

### 第三步：OWASP Top 10 安全检查

代码完成后执行安全检查：

| 编号 | 安全风险 | 检查要点 |
|------|----------|----------|
| A01 | 访问控制失效 | 身份认证、权限校验、会话管理 |
| A02 | 加密失败 | 敏感数据加密、传输安全、密钥管理 |
| A03 | 注入攻击 | SQL注入、XSS、命令注入防护 |
| A04 | 不安全设计 | 业务逻辑漏洞、权限边界设计 |
| A05 | 安全配置错误 | 默认配置、错误信息泄露、未授权功能 |
| A06 | 脆弱组件 | 第三方依赖版本、已知漏洞检查 |
| A07 | 身份认证失败 | 密码策略、多因素认证、会话失效 |
| A08 | 数据完整性失败 | 输入验证、签名校验、防篡改 |
| A09 | 日志监控失败 | 安全日志、异常告警、审计追踪 |
| A10 | SSRF | 外部资源访问控制、URL验证 |

### 第四步：代码审查

自审或请求审核：

| 审查维度 | 检查项 |
|----------|--------|
| 命名规范 | 变量、函数、类命名清晰一致 |
| 注释完整 | 关键逻辑有注释说明 |
| 代码复杂度 | 单函数不超过50行，嵌套不超过3层 |
| 异常处理 | 所有异常有明确处理逻辑 |
| 安全漏洞 | OWASP检查通过 |

### 第五步：输出交付物

生成以下交付物：

- 源代码 (src/)
- 单元测试代码 (tests/)
- 技术实现文档.md
- 代码审查报告.md
